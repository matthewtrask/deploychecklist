Deploychecklist

Make sure your next deploy goes smoothly.

This is aimed at helping developers new to deployment have a more successful deployment of code. Deploying can honestly be scary, and I hope this checkbox helps allievate the feeling. In this list you will find things that you might already do, things you didnt know you should do, and things to do should it go wrong. This can work if you are a team of two, or a team of 20, as long as you follow this list.

Somethings that are not on the list, yet need to be addressed is:

  • Development and Deployment Schedule
  • Deployment Time and Day
  • Who handles the actual deployment of code?
  • Who to alert (CEO, CTO, Product Owner, Dev Team, Infra Team if exists), and how to annouce when a deployment is made (Hipchat, Slack, Email)

Pre Pre Deploy

Prior to setting up for a pre deployment, what you should be looking for:

  • All PR's have been squashed to a single commit
  • Code has been reviewed by various team mates
  • Code was committed with tests to ensure the quality of code
  • Continious Integration has run and passed. (Travis, CircleCI, CodeShip, Jenkins)
  • PR has noted any changes to the database, migrations that need to be run, or places in the code base affected by the PR.

Pre Deploy

The next step, right before deployment, should have the following things take place:

  • Get code on a testing server, to test new code in the complete code base
  • Have both developers and non developers test. Compile results into a shared document and give developers time to fix anything that may have come up.
  • Restage changes, and do one more test through to see that all points were addressed
